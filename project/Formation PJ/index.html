HTML

HyperText Markup Language သို့မဟုတ် HTML သည် ဝဘ်ဘရောက်ဆာ တစ်ခုတွင် ပြသရန် ဒီဇိုင်းထုတ်ထားသော စာရွက်စာတမ်းများအတွက် စံ မှတ် ထားသော ဘာသာစကား ဖြစ်သည်။ Cascading Style Sheets (CSS) နှင့် JavaScript ကဲ့သို့သော scripting languages ​​ကဲ့သို့သော နည်းပညာများဖြင့် ၎င်းကို ကူညီပေး နိုင်ပါသည်။ ဝဘ်ဘရောက်ဆာ များသည် ဝဘ်ဆာဗာ တစ်ခုမှ သို့မဟုတ် ဒေသဆိုင်ရာ သိုလှောင်မှုမှ HTML စာရွက်စာတမ်းများကို လက်ခံရရှိ ပြီး စာရွက်စာတမ်းများကို မာလ်တီမီဒီယာ ဝဘ်စာမျက်နှာများအဖြစ် ပြန်ပေးသည် ။ HTML သည် ဝဘ်စာမျက်နှာ တစ်ခု၏ ဖွဲ့စည်းပုံကို အဓိပ္ပါယ် ရှိပြီး မူရင်းစာရွက်စာတမ်း၏ အသွင်အပြင်အတွက် လမ်းညွှန်ချက်များပါရှိသည်။ HTML ဒြပ်စင် များသည် HTML စာမျက်နှာများ၏ တည်ဆောက်မှုတုံးများဖြစ်သည်။ HTML တည်ဆောက်မှုများဖြင့်၊ ပုံများ နှင့် အပြန်အလှန်အကျိုးသက်ရောက်မှုပုံစံများ ကဲ့သို့သော အခြားအရာဝတ္ထုများကို ပြန်ဆိုထားသောစာမျက်နှာတွင် ထည့်သွင်းနိုင်ပါသည်။ HTML သည် ခေါင်းစဉ်များ၊ စာပိုဒ်များ၊ စာရင်းများ၊ လင့်ခ်များ ၊ ကိုးကားချက်များနှင့် အခြားအရာများ ကဲ့သို့ စာသားများအတွက် တည်ဆောက်ပုံ ဆိုင်ရာ ဝေါဟာရများကို ရည်ညွှန်းခြင်းဖြင့် တည်ဆောက်ပုံစာရွက်စာတမ်းများ ကို ဖန်တီးရန် နည်းလမ်းတစ်ခု ပံ့ပိုးပေးပါသည် ။ HTML ဒြပ်စင်များကို ထောင့်ကွင်း စကွက် များကို အသုံးပြု၍ တဂ်များ ဖြင့် ခွဲခြားသတ်မှတ်ထားသည် ။ ကဲ့သို့သော တဂ်များ နှင့် အကြောင်းအရာများကို စာမျက်နှာသို့ တိုက်ရိုက်မိတ်ဆက်ပါ။ တခြား tag တွေလိုမျိုးပေါ့။cစာရွက်စာတန်းများအကြောင်း အချက်အလက်များကို ဝိုင်းရံပြီး ပံ့ပိုးပေးကာ အခြားသော တဂ်များကို အစိတ်အပိုင်းခွဲများအဖြစ် ထည့်သွင်းနိုင်သည်။ ဘရောက်ဆာများသည် HTML တဂ်များကို မပြသသော်လည်း စာမျက်နှာ၏ အကြောင်းအရာကို အနက်ပြန်ဆိုရန် ၎င်းတို့ကို အသုံးပြုသည်။ HTML သည် ဝဘ်စာမျက်နှာများ၏ အပြုအမူနှင့် အကြောင်းအရာကို သက်ရောက်မှုရှိသော JavaScript ကဲ့သို့သော scripting language ဖြင့် ရေးသားထားသော ပရိုဂရမ်များကို မြှုပ်နှံနိုင်သည် ။ CSS ၏ ပါဝင်မှုသည် အကြောင်းအရာ၏ အသွင်အပြင်နှင့် အပြင်အဆင်ကို သတ်မှတ်သည်။ HTML ၏ထိန်းသိမ်းသူဟောင်းနှင့် CSS စံနှုန်းများကို လက်ရှိ ထိန်းသိမ်းသူဟောင်း World Wide Web Consortium (W3C) မှ ၁၉၉၇ ခုနှစ်ကတည်းက ရှင်းလင်းပြတ်သားစွာတင်ပြသည့် HTML ထက် CSS ကို အသုံးပြုရန် တိုက်တွန်းထားသည်။ [2] HTML5 ဟုလူသိများသော HTML ပုံစံ တစ်ခုကို အသုံးပြုသည်။ JavaScript နှင့် ပူးပေါင်း၍ အစိတ်အပိုင်း ကို အဓိကအသုံးပြု၍ ဗီဒီယိုနှင့် အသံကို ပြသပါ ။

Cascading Style Sheets ( CSS ) သည် HTML သို့မဟုတ် XML ကဲ့သို့သော markup ဘာသာစကား ဖြင့် ရေးသားထားသော စာရွက်စာတမ်း ၏ တင်ပြမှု ကို ဖော်ပြရန်အတွက် အသုံးပြု သည့် ပုံစံစာရွက်ဘာသာစကား တစ်ခု ( SVG ၊ MathML သို့မဟုတ် XHTML ကဲ့သို့ XML )။ [1] CSS သည် HTML နှင့် JavaScript တို့နှင့်အတူ World Wide Web ၏ အခြေခံအုတ်မြစ်နည်းပညာတစ်ခုဖြစ်သည် ။ [2] CSS သည် အပြင်အဆင် ၊ အရောင်များ နှင့် ဖောင့်များ အပါအဝင် အကြောင်းအရာနှင့် တင်ပြချက်များကို ပိုင်းခြား နိုင်စေရန် ဒီဇိုင်းထုတ်ထားသည် ။ [3] ဤခြားနားမှုသည် အကြောင်းအရာ သုံးစွဲနိုင် မှုကို တိုးတက်စေနိုင်သည် ။ တင်ဆက်မှုဝိသေသလက္ခဏာများ၏ သတ်မှတ်ချက်များတွင် ပိုမိုပြောင်းလွယ်ပြင်လွယ်နှင့် ထိန်းချုပ်မှုကို ပေးဆောင်ပါ။ သီးခြား .css ဖိုင်တွင် သက်ဆိုင်ရာ CSS ကို သတ်မှတ်ခြင်းဖြင့် ဖော်မတ်ချခြင်းကို မျှဝေရန် ဝဘ်စာမျက်နှာ များစွာကို ဖွင့် ပေးသည်၊ ၎င်းသည် ဖွဲ့စည်းတည်ဆောက်ပုံဆိုင်ရာ အကြောင်းအရာတွင် ရှုပ်ထွေးမှုနှင့် ထပ်တလဲလဲမှုကို လျှော့ချပေးသည်။ ဖိုင်နှင့် ၎င်း၏ဖော်မတ်ချသည့် စာမျက်နှာများကြား စာမျက်နှာ load speed ကို မြှင့်တင်ရန် .css ဖိုင်ကို ကက်ရှ် လုပ်ရန် ဖွင့်ပါ။ ဖော်မတ်ချခြင်းနှင့် အကြောင်းအရာကို ပိုင်းခြားထားခြင်းသည် မျက်နှာပြင်ပေါ်ရှိ၊ ပုံနှိပ်ခြင်း၊ အသံဖြင့် (စကားသံအခြေခံသည့်ဘရောက်ဆာ သို့မဟုတ် စခရင်ဖတ်စက် ) နှင့် မျက်မမြင်စာ ကြည့် စနစ်ကဲ့သို့သော ကွဲပြားသော rendering နည်းလမ်းများအတွက် တူညီသော markup စာမျက်နှာကို ကွဲပြားသောပုံစံများဖြင့် တင်ပြနိုင်စေသည် ။ ထိတွေ့နိုင်သော ကိရိယာများ။ CSS တွင် အကြောင်းအရာကို မိုဘိုင်းကိရိယာတစ်ခု ပေါ်တွင် ဝင်ရောက်အသုံးပြုပါက အစားထိုးဖော်မတ်ချခြင်းအတွက် စည်းမျဉ်းများ ပါရှိသည်။ [4] စည်းမျဥ်း တစ်ခုထက်ပို၍ သီးခြားဒြပ်စင်တစ်ခုနှင့် ကိုက်ညီပါက မည် သည့်စတိုင်စည်းမျဉ်းကို ကျင့်သုံးမည်ကို ဆုံးဖြတ်ရန် သတ်မှတ်ထားသော ဦးစားပေး အစီအစဉ်မှ လာပါသည်။ ဤဦးစားပေးအစီအစဉ်သည် ကြိုတင်မှန်းဆနိုင်သည်။ CSS သတ်မှတ်ချက်များကို World Wide Web Consortium (W3C) မှ ထိန်းသိမ်းထားသည်။ အင်တာနက်မီဒီယာအမျိုးအစား ( MIME အမျိုးအစား ) text/cssကို RFC 2318 (မတ်လ 1998) ဖြင့် CSS ဖြင့် အသုံးပြုရန်အတွက် မှတ်ပုံတင်ထားသည်။ W3C သည် CSS စာရွက်စာတမ်းများအတွက် အခမဲ့ CSS အတည်ပြုခြင်းဝန်ဆောင်မှု ကို လုပ်ဆောင်သည်။ [5] HTML အပြင် အခြားသော markup ဘာသာစကားများသည် XHTML ၊ ရိုးရိုး XML ၊ SVG နှင့် XUL အပါအဝင် CSS ကို အသုံးပြုမှုကို ပံ့ပိုး ပေးပါသည်။

JavaScript ( / ˈ dʒ ɑː v ə s k r ɪ p t / ) သည် JS ဟု အတိုကောက်ခေါ်လေ့ရှိပြီး World Wide Web ၏ အဓိကနည်းပညာများထဲမှ တစ်ခုဖြစ်သည့် HTML နှင့် CSS တို့နှင့်အတူ ပရိုဂရမ်းမင်းဘာသာစကား ဖြစ်သည်။ 2022 ခုနှစ်အထိ၊ ဝဘ်ဆိုဒ် များ၏ 98% သည် ဝဘ်စာမျက်နှာ အပြုအမူအတွက် ကလိုင်းယင့်ဘက်မှ JavaScript ကို အသုံးပြုကြပြီး မကြာခဏ ပြင်ပ အဖွဲ့အစည်း စာ ကြည့် တိုက် များကို ထည့်သွင်းထားသည် ။ အဓိက ဝဘ်ဘရောက်ဆာ များအားလုံး တွင် ၎င်းကိုလုပ်ဆောင်ရန် သီးခြား JavaScript အင်ဂျင် တစ်ခုရှိသည် ။သုံးစွဲသူများ ၏ စက်များတွင် ကုဒ် ။ JavaScript သည် ECMAScript စံနှုန်း နှင့် အ ညီ မကြာခဏ အချိန်နှင့်တပြေးညီ စုစည်းထားသော ဘာသာစကားတစ်ခုဖြစ်သည် ။ [10] ၎င်းတွင် ဒိုင်းနမစ်စာရိုက်ခြင်း ၊ ရှေ့ပြေးပုံစံအခြေခံ အရာဝတ္ထု-ဦးတည် ချက်နှင့် ပထမတန်းစားလုပ်ဆောင်ချက်များ ပါရှိသည်။ ၎င်းသည် ဘက်စုံပါရာဒိုင်း ၊ ဖြစ်ရပ်မောင်းနှင်မှု ၊ လုပ်ဆောင်နိုင်မှု ၊ နှင့် မရှိမဖြစ် ပရိုဂရမ်းမင်းပုံစံများ ကို ပံ့ပိုးပေးသည် ။ ၎င်းတွင် စာသား၊ ရက်စွဲများ၊ ပုံမှန်အသုံးအနှုန်းများ ၊ စံ ဒေတာတည်ဆောက်ပုံများ ၊ နှင့် Document Object Model တို့နှင့် အလုပ်လုပ်ရန်အတွက် အ ပလီကေးရှင်း ပရိုဂရမ်းမင်း အင်တာဖေ့ စ်များ (APIs) ပါရှိသည်။ (DOM)။ ECMAScript စံနှုန်းတွင် ကွန်ရက်ချိတ်ဆက်မှု ၊ သိုလှောင်မှု သို့မဟုတ် ဂရပ်ဖစ် ပံ့ပိုးမှုများ ကဲ့သို့သော မည်သည့် အဝင်/အထွက် (I/O) မပါဝင်ပါ ။ လက်တွေ့တွင်၊ ဝဘ်ဘရောက်ဆာ သို့မဟုတ် အခြား runtime စနစ်သည် I/O အတွက် JavaScript API များကို ပံ့ပိုးပေးသည်။ JavaScript အင်ဂျင် များကို မူလက ဝဘ်ဘရောက်ဆာများတွင်သာ အသုံးပြုခဲ့သော်လည်း ယခုအခါ အချို့သော ဆာဗာများ နှင့် အ ပလီကေးရှင်း အမျိုးမျိုး၏ အဓိက အစိတ်အပိုင်းများဖြစ်သည် ။ ဤအသုံးပြုမှုအတွက် ရေပန်းအစားဆုံး runtime စနစ်မှာ Node.js ဖြစ်သည်။ Java နှင့် JavaScript တို့သည် အမည်၊ အထားအ သို နှင့် သက်ဆိုင်ရာ စံပြစာကြည့်တိုက် များတွင် ဆင်တူ သော်လည်း ဘာသာစကားနှစ်ခုသည် ကွဲပြားပြီး ဒီဇိုင်းတွင် များစွာကွဲပြားပါသည်။
